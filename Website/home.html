General Task

In this task a WebGIS Application for crisis routing should be developed. Therefore different services, like "OSRM", "pgRouting" or "BRouter", can be used for. Besides making some thoughts about the best service to use for this project, also a decision has to be done, what kind of vehicle(s) should be chosen for this routing activities and on which region the project has to be focused on. Independent of the service always "OpenStreetMap"-data has to be implemented.

Regarding the service finally "BRouter" has been chosen for this task. The reasons for this decision can be seen within the description about "BRouter" right below (see "Used Service").Talking about the vehicles first of all cars should be adopted into our project, but besides that also a routing description for cyclists and pedestrians might be necessary, in case of destroyed streets, where cars can´t drive on anymore. For the map data the little country of Andorra has been chosen, because therefore not so much data has to be stored, so that the work isn´t that time-consuming.

Beside the normal routing functionalities for this kind of vehicles an additional functionality should be added later on. This additional functionality is called "Blocked areas" and should be added due to the fact, that in this case an application for crisis routing should be established. With that the appropriate people are warned just in time, if a street or a way is impassable, so that an alternative route can be calculated rapidly.

This project was published by Dr. Marco Lechner within the lecture "OpenSource GIS" at the "Hochschule Karlsruhe Technik und Wirtschaft". It should be done in form of a project by five students studying "Geomatics" there.

Used Service

Definition
BRouter is a routing engine designed to calculate optimal cycling routes using OpenStreetMap and elevation data. The programming language of this kind of service is Java and there´s existing an additional online version with the name "brouter-web". In case that the browser can´t run "brouter-web" there´s also a simple old online version as a fallback. But also an Android app for offline routing is available from "Google-Play".

Comparison with other services
The service BRouter uses freely configurable routing profiles and works fully offline on any Android phone. It is interfaced to some of the most popular Android map tools. Besides that BRouter uses a sophisticated routing-algorithm with elevation consideration and it also allows alternative route calculations. Additionally the service supports via-points, as well as no-go-points. It can consider long distance cycle routes and the routing-data is available worldwide with automatic weekly updates.

Used data
Thanks to "OpenStreetMap.org" and contributors BRouter uses user-generated, collaboratively collected free geodata, along with SRTM for terrain data. The service uses highly configurable routing profiles which may be selected or customised. Tracks are just calculated as GPX-Files, which means that it doesn´t display any map. It also doesn´t give any navigation instructions. So in order for BRouter to be useful a map-tool is needed. Currently BRouter cooperates with three different maptools, that are called "OsmAnd", "Locus Map" and "Oruxmaps". The appropriate user has to install at least one of those maptools and has to get familiar with that. The BRouter-App can be either installed from "Google Play Store", "F-Droid" or directly from the website (the APK-File is contained within the "brouter_1_4.zip" distribution zip-file). After accepting a base-directory proposal, BRouter creates some subfolders, that are relative to this base directory.
